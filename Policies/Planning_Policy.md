PLANNING POLICY  
Version: 1.0   
Approval Date: 7th July, 2026     
Owner: Information Security Officer (ISO)  
Classification: Internal / Restricted  
________________________________________
1. Purpose  
The purpose of this Planning Policy is to establish requirements for developing, documenting, and maintaining security plans that define how Veltrix Pay information systems implement and maintain security controls in alignment with the Risk Management Framework (RMF).  
________________________________________
2. Scope  
This policy applies to:  
•	All Veltrix Pay information systems  
•	All personnel involved in system design, development, implementation, and operation  
•	All environments hosting or processing organizational information (on-premises, cloud, hybrid)  
•	All system owners, administrators, and security personnel  
________________________________________  
3. Policy Statements  
3.1 Security Planning Requirements  
•	Each information system must have a documented System Security Plan (SSP).  
•	The SSP must describe system components, operational environment, implemented security controls, and known risks.  
•	Security plans must be developed before system deployment and updated throughout the system lifecycle.  
•	The payment switch and ledger systems require an SSP addendum addressing PCI DSS scope boundaries.  
________________________________________
3.2 Plan Approval  
•	The SSP must be reviewed and approved by:  
o	System Owner  
o	Information Security Officer (ISO)  
o	Authorizing Official (AO)  
________________________________________
3.3 Security Plan Maintenance  
•	SSPs must be updated to reflect:  
o	System changes (hardware, software, architecture, or data flows)  
o	New or modified security controls  
o	Results from assessments or monitoring activities  
•	SSPs must undergo a formal review at least annually.  
________________________________________
3.4 System Life-Cycle Planning  
•	Security considerations must be integrated into each phase of the system development life cycle (SDLC).  
•	All technology acquisitions must include security and privacy requirements.  
•	Major redesigns or migrations require updated planning documentation and new risk assessments.  
________________________________________
3.5 Resource Planning  
•	System owners must identify and allocate resources necessary to implement, maintain, and monitor security controls.  
•	Resource planning must consider:  
o	Personnel  
o	Training  
o	Tools and technologies  
o	Budget requirements    
________________________________________
3.6 Security Architecture Documentation  
•	System architecture must be documented and maintained, including:  
o	Network diagrams  
o	Data flow diagrams (including cardholder data flow per PCI DSS Requirement 1)  
o	System boundary definitions  
o	Integration with external systems (banks, card networks, payment processors)  
•	Architectural diagrams must be updated as systems evolve.  
________________________________________
4. Roles & Responsibilities  
Role	Responsibilities  
System Owner	Develops and maintains SSPs, ensures changes are documented.  
ISO	Provides oversight, approves security plans, ensures alignment with RMF.  
Authorizing Official (AO)	Reviews and approves plans as part of the authorization process.  
IT Engineers & Developers	Provide technical details and assist with architecture documentation.  
Security Team	Supports control selection, ensures plans meet compliance requirements.  
________________________________________
5. Compliance  
Non-compliance with this policy may result in delays in system authorization, suspension of operations, or disciplinary action as determined by organizational leadership.
