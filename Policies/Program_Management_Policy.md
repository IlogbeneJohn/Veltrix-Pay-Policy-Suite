Owner: Program Manager (PM) & Information Security Officer (ISO)   
Classification: Internal / Restricted  
________________________________________
1. Purpose  
The purpose of this Program Management Policy is to establish a framework for managing, overseeing, and governing Veltrix Pay's information security and risk management program in accordance with NIST RMF and applicable regulatory requirements, including CBN cybersecurity guidelines for payment service providers.  
________________________________________
2. Scope  
This policy applies to:  
•	All organizational programs and business units  
•	All information systems within Veltrix Pay's authorization boundary  
•	All employees, contractors, and third parties who participate in program oversight, governance, or risk management activities  
________________________________________
3. Policy Statements  
3.1 Governance Structure  
•	A formal information security governance structure must be established and maintained.  
•	Roles and responsibilities for managing the security and risk program must be clearly defined and documented.  
•	The Program Manager (PM) ensures alignment between security requirements and organizational mission.  
•	A Risk and Information Security Committee, including Executive and Board representation, must convene quarterly.  
________________________________________
3.2 Risk Management Integration  
•	Risk management activities must be integrated into organizational planning, budgeting, and system development.  
•	Risk decisions must be documented and approved by appropriate authorizing officials.  
•	The organization must maintain a continuous risk monitoring process.  
________________________________________
3.3 Resource Allocation  
•	Adequate personnel, training, and technical resources must be allocated to support the security program.  
•	Program funding must be reviewed annually to ensure continued ability to manage security risks.  
________________________________________
3.4 Roles & Responsibilities  
•	Program Manager (PM): Oversees execution of the security program and ensures compliance with this policy.  
•	Information Security Officer (ISO): Provides strategic security guidance and program-level oversight.  
•	System Owners: Implement security controls within their systems and support program objectives.  
•	Executives / Leadership: Approve risk decisions and provide strategic direction.  
________________________________________
3.5 Strategic Planning  
•	The organization must maintain a documented Information Security Program Plan (ISPP).  
•	Program goals and objectives must be reviewed and updated annually.  
•	Security initiatives must be aligned with organizational mission needs and CBN regulatory expectations for payment service providers.  
________________________________________
3.6 Security Performance Metrics  
•	The program must track and report security performance metrics, including:  
o	Control effectiveness  
o	Compliance status  
o	Vulnerability trends  
o	Incident response metrics  
o	Fraud loss ratios and transaction monitoring effectiveness  
•	Metrics must be reviewed by leadership at least quarterly.  
________________________________________
3.7 Continuous Improvement  
•	The organization must continuously assess and improve its security posture.  
•	Lessons learned from audits, incidents, and assessments must feed into program enhancements.  
________________________________________
3.8 Documentation Requirements  
The program must maintain the following documentation, at minimum:  
•	Information Security Program Plan (ISPP)  
•	Risk Management Strategy  
•	Security Assessment Plan & Reports  
•	Continuous Monitoring Strategy    
•	LTAs, POA&Ms, and remediation plans  
________________________________________
4. Compliance  
Failure to comply with this policy may result in disciplinary action and may expose Veltrix Pay to security risks and regulatory non-compliance.  
________________________________________
5. Review & Updates  
This policy must be reviewed annually and updated as needed to reflect changes in organizational priorities, regulatory requirements, or risk environment.
